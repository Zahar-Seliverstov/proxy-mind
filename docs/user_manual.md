# Руководство пользователя ProxyMind

**Продукт:** ProxyMind — веб-интерфейс управления AI-ассистентами кода  
**Версия:** 1.0

---

## 1. Назначение

ProxyMind — веб-приложение, которое автоматизирует работу с CLI AI-ассистентами (Claude Code, Aider и др.), запущенными в терминальных сессиях tmux. Пользователь формулирует задачу в браузере, система разбивает её на шаги и последовательно выполняет их через AI-ассистент без ручного вмешательства.

---

## 2. Требования

- **tmux** запущен (`tmux new -s имя` или любая существующая сессия)
- **Ollama** запущен и доступен (`http://localhost:11434` по умолчанию)
- В хотя бы одной панели tmux запущен AI-ассистент (Claude Code, Aider и т.д.)
- Браузер (Chrome, Firefox, любой современный)

---

## 3. Запуск приложения

Открыть два терминала (или tmux-окна):

**Терминал 1 — backend:**
```bash
cd /путь/к/nmnt/backend
./start.sh
```

**Терминал 2 — frontend:**
```bash
cd /путь/к/nmnt/frontend
npm run dev
```

После этого открыть браузер: **http://localhost:5173**

---

## 4. Интерфейс

### 4.1. Боковая панель (SessionsPanel)

Левая колонка показывает дерево tmux-сессий: **Сессия → Окно → Панель**.

- **Раскрытие:** кликнуть на сессию или окно
- **Выбор панели:** кликнуть на нужную панель — она подсветится синим, рабочая область справа активируется
- **Создание сессии:** нажать кнопку «+» → ввести имя и рабочий каталог → «Создать»
- **Удаление:** нажать «×» рядом с нужным элементом
- **Настройки:** кнопка шестерёнки в нижней части панели

### 4.2. Рабочая область (PaneWorkspace)

Главная зона работы. Отображается после выбора панели.

**Верхняя строка:**
- **Выбор режима** (ModeSelect) — слева
- **Выбор модели Ollama** (ModelSelect) — справа

**Поле ввода промпта** — многострочное, поддерживает любой текст на русском или английском.

**Кнопка Generate** — запускает обработку промпта.

---

## 5. Режимы работы

### Auto Plan (рекомендуется для новых задач)
AI разбивает задачу на 3–8 последовательных шагов. Перед генерацией задаёт уточняющие вопросы. После — показывает план на редактирование.

### Rewrite
AI переформулирует ваш промпт чётко и однозначно, сохраняя все детали. Результат — один промпт, который отправляется напрямую.

### Direct
Промпт переводится на английский язык и отправляется AI-ассистенту как есть, без каких-либо изменений.

### Manual Plan
Вы сами пишете шаги (по одному на строку или нумерованным списком). Система переводит их на английский и выполняет по очереди.

---

## 6. Процесс выполнения задачи (Auto Plan)

### Шаг 1: Ввод промпта
Напишите задачу в поле ввода. Пример: *«Создать Telegram-бота, который присылает погоду по запросу»*

### Шаг 2: Диалог уточняющих вопросов
Система задаёт вопросы по одному. Для каждого предлагает 3–4 варианта ответа или кнопку «Skip» (пропустить). Выберите вариант или напишите свой ответ.

### Шаг 3: Ревью плана
После сбора ответов AI генерирует план. Вы можете:
- **Редактировать** любой шаг (кликнуть на текст)
- **Удалить** шаг (кнопка «×»)
- **Регенерировать** весь план (кнопка «Regenerate»)
- **Запустить** (кнопка «Run»)

### Шаг 4: Мониторинг (RunMonitor)
После запуска открывается монитор:
- **Список шагов** с иконками статусов (⏳ ожидание, 🔄 выполняется, ✅ готово, ❌ ошибка)
- **Лог событий** — в реальном времени показывает действия оркестратора
- **Кнопки управления:**
  - «Pause» — приостановить выполнение
  - «Resume» — продолжить после паузы
  - «Cancel» — отменить запуск

---

## 7. Автоответы оркестратора

В процессе работы AI-ассистент может задавать вопросы:
- **Yes/No** (подтверждение действия) — оркестратор отвечает автоматически
- **Выбор из меню** — оркестратор выбирает вариант, соответствующий цели шага
- **Произвольный текст** — оркестратор формулирует короткий ответ

Если один и тот же вопрос повторяется 5 раз — система считает это зависанием и переходит в статус ошибки.

---

## 8. Настройки

Открываются кнопкой шестерёнки в SessionsPanel.

| Параметр | Описание | По умолчанию |
|---|---|---|
| Ollama Base URL | Адрес Ollama API | http://localhost:11434 |
| Telegram Bot Token | Токен бота для уведомлений | (пусто) |
| Telegram Chat ID | ID чата для уведомлений | (пусто) |

Нажать «Save» для сохранения.

---

## 9. Уведомления

- В правом верхнем углу отображаются toast-уведомления (success/info/warn/error) — исчезают автоматически
- Панель уведомлений (иконка колокольчика) — история всех уведомлений, счётчик непрочитанных
- Если настроен Telegram-бот, уведомления о завершении задач дублируются в Telegram
