# Протокол тестирования ProxyMind

**Продукт:** ProxyMind — веб-интерфейс управления AI-ассистентами кода  
**Версия:** 1.0 (преддипломная практика)  
**Дата тестирования:** 09–15 мая 2026 г.  
**Тестировщик:** Селиверстов З.А.  
**Окружение:** Linux (CachyOS), tmux 3.x, Ollama qwen2.5-coder:14b, Python 3.14, Node.js 22

---

## Инструменты тестирования

- **tools/fake_claude.py** — симулятор CLI-ассистента для воспроизведения сценариев без реального AI
- **Браузер** (Chromium/Firefox) — для тестирования веб-интерфейса
- **Claude Code** — реальный CLI-ассистент для интеграционного тестирования

## Тестирование безопасности

| ID | Проверка | Результат |
|---|---|---|
| TS-001 | Backend слушает только 127.0.0.1:3000 (не доступен из сети) | Подтверждено: uvicorn bind 127.0.0.1 |
| TS-002 | Входные данные промптов валидируются Pydantic-схемами (типы, обязательные поля) | Подтверждено: schemas/ai_schema.py |
| TS-003 | Токены Telegram хранятся только в файле настроек пользователя (~/.config/proxymind/) | Подтверждено: файл вне репозитория |
| TS-004 | CORS не открыт публично (Vite-прокси работает только в dev-режиме) | Подтверждено |
| TS-005 | Команды в tmux передаются через libtmux API (нет интерполяции shell-команд) | Подтверждено: send_text не вызывает shell |
